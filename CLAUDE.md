# fazafilms-site — сайт студии «Фаза» (гид для Claude Code)

Статический сайт студии видеопродакшна «Фаза». Хостинг — **GitHub Pages** (домен `fazafilms.ru`).
Часть проекта FazaFlow (внутренняя система — отдельный репозиторий `Mak207/fazaflow`).
**Прод живой.** Работаем **через ветки и Pull Request**, не пушим напрямую в `main`.

## Структура
- `index.html` — вся разметка (hero, портфолио-грид, услуги, форма «Расскажите о задаче», футер).
- `css/styles.css` — стили (тёмная кинематографичная тема, бренд Фазы).
- `js/main.js` — интерактив: рендер портфолио, hover-видео, hero-видео с ротацией, форма брифа.
- `js/selection.js` — данные портфолио: массив из 253 объектов `{id,title,vimeo,poster,cat,clip_s3,...}`.
- `assets/` — `hero-*.mp4` (самохост-шоурилы), `posters/<id>.webp` (постеры), `previews/<id>.mp4` (hover-видео 640px), `hero.webp` (фолбэк).
- `CNAME` — `fazafilms.ru`.

## Как устроено (важное)
- **Hero — самохостящееся видео** `<video>` (`assets/hero-*.mp4`), НЕ Vimeo-iframe: ролики студии на Vimeo имеют доменное ограничение встраивания → отдают 401. Ротация ролика — через `localStorage` (разный при каждом визите). Постер-фолбэк на мобильных/reduce-motion.
  - ⚠️ анимация фона на `inset:0` элементах — зум **только scale**, без `translate(-50%,-50%)` (иначе видео/постер съезжает на пол-экрана).
- **Портфолио** — рендер из `selection.js`, **новые первыми** (сорт по vimeo-id убыв.), hover-видео грузится лениво при наведении (десктоп).
- **Форма «Расскажите о задаче»** — текст ИЛИ запись голоса (MediaRecorder) → POST на `https://panel.fazafilms.ru/api/brief` (бэкенд FazaFlow, CORS только с fazafilms.ru) → бот расшифровывает → бриф в Kitsu + уведомление руководителю. Анти-спам: капча (`/api/captcha`, математика+HMAC) + honeypot + лимит по IP.

## Деплой
Правишь → ветка → PR → ревью владельца → merge в `main` → **GitHub Pages пересобирает (~1 мин)**.
Большие медиа (видео/постеры) оптимизируй перед коммитом (репо не должен пухнуть без нужды). Исходники роликов/нарезок — в S3 (`faza/video/`, `faza/cuts/`), не в репо.

## Проверка результата
Перед «готово» — посмотри сайт глазами (скриншот headless на десктопе И на 390px-мобиле). На headless секции стартуют с `opacity:0` (анимация появления) — для скриншота добавляй `--force-prefers-reduced-motion`, иначе чёрный экран. DNS со станции может «врать» → форсируй IP GitHub Pages (`--host-resolver-rules`).

Бэкенд формы и трекинг живут в репозитории `Mak207/fazaflow` (панель `panel/app.py` на `panel.fazafilms.ru`). Полная карта проекта и доступы — см. онбординг-документ у владельца.
